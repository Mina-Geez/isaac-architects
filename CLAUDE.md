# Isaac Architects — project notes

Rebrand of **IDEA Designs** (Eng. Mina Kamal Isaac, Alexandria, Egypt) into **Isaac Architects**. Portfolio website + 18 dedicated project pages.

## Deployment

- **Repo**: https://github.com/Mina-Geez/isaac-architects (public)
- Deploy: push to `main` → **Cloudflare** rebuilds/serves automatically (see `.remember`/memory).
- **Canonical domain** is a single setting: `content/settings.toml` → `canonical_base`. Every `canonical`/`og:url`/`og:image`/sitemap/robots/JSON-LD URL derives from it. Set it to the real custom domain (currently a placeholder).

## Content engine (CMS)

The site is generated by a small Python engine — **not** edited by hand. Content lives in `content/`; `python build.py` renders everything to the repo root. See `README.md` and `docs/superpowers/specs/2026-06-14-content-engine-design.md`.

- `content/projects/<slug>/index.md` — one folder per project: TOML frontmatter (`+++…+++`) + Markdown body, with `hero.jpg` and gallery `NN.jpg` colocated; gallery declared via `[[gallery]]` (`src` + `alt`). **Edit projects here**, then `python build.py`.
- `content/settings.toml` — canonical domain, contact info, SEO descriptions.
- `templates/home.html`, `templates/project.html` — Jinja templates (home copy is inline; the project grid is generated from `content/projects/`).
- `static/site.css`, `static/project.css`, `static/site.js` — sources; the build emits content-hashed copies into `static/` and links them. **Edit CSS/JS here**, not in the generated HTML.
- `engine/` — parsers, validators, image/SEO/security/asset helpers (unit-tested in `tests/`).
- `build.py` — the entrypoint. Run it after any content/template/CSS change.
- **Generated (do not edit):** `index.html`, `projects/*.html`, `images/*`, `static/site.<hash>.*`, `sitemap.xml`, `robots.txt`, `_headers`, `.well-known/security.txt`.
- `images/og-image.jpg` — 1200×630 social-share image (static).
- `brand/*.svg` — official logos. **Preserve exactly.** Reference, don't re-inline.
- `brand/isaac-architects-brand-system.pdf` — brand guidelines (has a real text layer; extract with `PYTHONIOENCODING=utf-8`).
- `tools/` — `import_legacy.py` (one-time content import), `parity_shots.py`/`parity_diff.py` (visual-parity gate), `crop_project_images.py` + `_legacy/` (retired generator + `gallery_data.json`, kept for reference).
- `Client input - previous projects/`, `pdf_pages/` — source materials. Gitignored.

## Build flow

```
content/  ──>  python build.py  ──>  index.html + projects/*.html + images/* +
              static/site.<hash>.* + sitemap.xml + robots.txt + _headers + security.txt
```

Claude Code commands: `/add-project`, `/edit-project`, `/new-page`, `/publish`.
Before merging a content/template change, run the parity gate: `python tools/parity_shots.py tools/_parity/after . && python tools/parity_diff.py`.

## Responsive breakpoints

`1024px` (tablet — 2-col grids) → `768px` (phone — single col, hamburger menu) → `480px` (small phone — tightest spacing). Rules live in `static/site.css` (home) and `static/project.css` (project pages) at all three.

## Working with the PDFs

Source PDFs have no text layer. Don't try to Read them directly or extract text via PowerShell — both produce empty output. Use the pre-rendered PNGs in `pdf_pages/`, or re-render new files with `pymupdf` at 2× resolution (`page.get_pixmap(matrix=fitz.Matrix(2,2))`).
