## Deployment

- **Repo**: https://github.com/Mina-Geez/isaac-architects (public). Worker: `isaac-architects` (Cloudflare Workers static assets). Live: **isaacarchitects.com**.
- Deploy: push to `main` → Cloudflare auto-deploys **`website/`** (per `wrangler.jsonc` `assets.directory = "./website"`, `not_found_handling: "404-page"`).
- **Canonical domain** is one setting: `website/content/settings.toml` → `canonical_base = "https://isaacarchitects.com"`. All canonical/og/sitemap/JSON-LD URLs (extensionless) derive from it.

## Content engine (CMS) — under `website/`

Generated by a small Python engine — **not** edited by hand. Run `python website/build.py` (paths resolve from the engine's own location, so cwd doesn't matter). See `website/README.md` and `website/docs/superpowers/specs/2026-06-14-content-engine-design.md`.

- `website/content/projects/<slug>/index.md` — one folder per project: TOML frontmatter + Markdown body, `hero.jpg` + gallery `NN.jpg` colocated, gallery via `[[gallery]]` (`src`+`alt`). **Edit projects here**, then build.
- `website/content/settings.toml` — canonical domain, contact, SEO.
- `website/templates/*.html`, `website/static/{site,project}.css` + `site.js` — sources (build emits content-hashed copies). **Edit CSS/JS here**, not the generated HTML.
- `website/engine/` — parsers/validators/image/SEO/security/asset helpers (tests in `website/tests/`).
- **Brand logos**: `build.py` copies the served SVGs from the shared `brand/` into `website/brand/` so the site is self-contained. The workspace `brand/` stays the **source of truth** — edit logos there, then rebuild.
- **Generated (do not edit):** `website/{index,404}.html`, `website/projects/*.html`, `website/images/*`, `website/static/site.<hash>.*`, `website/{sitemap.xml,robots.txt,_headers}`, `website/.well-known/security.txt`, `website/brand/*.svg`.
- `website/Client input - previous projects/`, `website/pdf_pages/` — source materials. Gitignored + `.assetsignore`-excluded.

## Responsive breakpoints

`1024px` (tablet — 2-col grids) → `768px` (phone — single col, hamburger menu) → `480px` (small phone — tightest spacing). Rules live in `static/site.css` (home) and `static/project.css` (project pages) at all three.

## Working with the PDFs

Source PDFs have no text layer. Don't try to Read them directly or extract text via PowerShell — both produce empty output. Use the pre-rendered PNGs in `pdf_pages/`, or re-render new files with `pymupdf` at 2× resolution (`page.get_pixmap(matrix=fitz.Matrix(2,2))`).
