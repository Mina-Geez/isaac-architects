# Isaac Architects — project notes

Rebrand of **IDEA Designs** (Eng. Mina Kamal Isaac, Alexandria, Egypt) into **Isaac Architects**. Portfolio website + 18 dedicated project pages.

## Deployment

- **Repo**: https://github.com/Mina-Geez/isaac-architects (public)
- **Live URL** (GitHub Pages, served from `main` branch root): https://mina-geez.github.io/isaac-architects/
- All `og:url`, `og:image`, `canonical` meta tags are hardcoded against this URL. If a custom domain is added later, those tags need updating in `index.html` and the `<head>` block inside `generate_project_pages.py`.

## Files

- `index.html` — the home page (was `isaac-architects-website.html` before the GitHub Pages rename). Single-file HTML linking to `brand/*.svg`.
- `projects/<slug>.html` — 18 dedicated project pages. **Do not edit by hand** — they are regenerated by `generate_project_pages.py`.
- `images/<slug>.jpg` — 18 hero images (one per project, used on home grid and project hero).
- `images/<slug>-N.jpg` — gallery images (up to 8 per project), shown on each project page's gallery section.
- `images/og-image.jpg` — 1200×630 social-share image.
- `brand/logo-mark.svg`, `brand/logo-lockup-dark.svg`, `brand/logo-lockup-light.svg` — official logos. **Preserve exactly.** Reference from HTML (don't re-inline — bloats file from ~25KB to ~1.2MB).
- `brand/isaac-architects-brand-system.pdf` — brand guidelines.
- `gallery_data.json` — workflow output: per-project list of `{source_file, crop_*_pct, quality, description}`. Drives the gallery cropping.
- `generate_project_pages.py` — **the build script**. Reads `gallery_data.json`, crops every gallery image to `images/<slug>-N.jpg`, then renders all 18 project pages. Re-run after any project metadata or gallery change.
- `crop_project_images.py` — older script that produced the 18 hero images. Hero images live in `images/<slug>.jpg`. Keep around for re-cropping heroes if needed.
- `Client input - previous projects/`, `pdf_pages/` — source materials. Gitignored (large, regeneratable / private).

## Build flow

```
gallery_data.json  ──┐
                     ├──>  python generate_project_pages.py  ──>  images/<slug>-N.jpg + projects/*.html
PROJECTS dict in    ─┘
generate_project_pages.py
```

To regenerate hero images: re-run `crop_project_images.py` (needs `pdf_pages/` populated).

## Responsive breakpoints

`1024px` (tablet — 2-col grids) → `768px` (phone — single col, hamburger menu) → `480px` (small phone — tightest spacing). Both `index.html` and the project-page template in `generate_project_pages.py` have rules at all three.

## Working with the PDFs

Source PDFs have no text layer. Don't try to Read them directly or extract text via PowerShell — both produce empty output. Use the pre-rendered PNGs in `pdf_pages/`, or re-render new files with `pymupdf` at 2× resolution (`page.get_pixmap(matrix=fitz.Matrix(2,2))`).
